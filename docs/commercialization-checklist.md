# 商用化清单

本文档用于梳理“记账项目”从前端原型升级到可商用产品还需要补齐的能力，按优先级拆分为 `P0 / P1 / P2`。

当前判断：项目已具备原型演示和静态上线能力，但距离可稳定对外商用还有明显差距，主要短板集中在数据持久化、账号权限、安全合规、后端能力和工程化保障。

## 当前阶段定位

- 当前形态：纯静态前端原型
- 部署方式：GitHub Pages
- 数据存储：浏览器本地 `localStorage`
- 适合用途：演示、内部试用、需求验证、交互确认
- 不适合用途：真实生产记账、多人协作、长期数据留存、正式对外收费服务

## P0：商用前必须完成

这些事项不补齐，项目不具备正式上线条件。

### 1. 后端与数据库

- 建立正式后端服务，至少覆盖债务记录、还款计划、付款记录、附件、导入草稿
- 引入数据库，支持持久化存储和基础数据约束
- 设计数据模型、唯一键、更新时间、创建人、删除标记
- 支持数据备份、恢复和基础迁移方案

### 2. 账号体系与权限控制

- 支持注册、登录、退出登录、重置密码
- 支持手机号或邮箱验证
- 支持组织、成员、角色、权限模型
- 区分查看、编辑、删除、导出、审批等操作权限

### 3. 数据安全

- 服务端鉴权和接口权限校验
- 敏感信息传输加密
- 核心数据存储加密或脱敏策略
- 审计日志，记录谁在什么时间修改了什么数据
- 防止未授权访问、越权操作和批量爬取

### 4. 文件与附件能力

- 引入正式对象存储保存凭证附件
- 限制上传类型、大小和数量
- 增加文件安全校验和异常文件拦截
- 附件访问权限控制
- 附件和业务记录建立稳定关联关系

### 5. 基础业务闭环

- 定义债务、应收、应还、逾期、已结清、部分还款等状态机
- 保证还款记录与剩余金额一致
- 支持撤销、冲正、修改留痕
- 处理币种、金额精度、日期边界和异常输入

### 6. 基础工程保障

- 拆分当前单文件脚本，做模块化改造
- 建立测试体系，至少覆盖核心业务逻辑
- 增加错误日志、运行监控和异常告警
- 区分开发、测试、生产环境

### 7. 法务与合规基础

- 补齐用户协议
- 补齐隐私政策
- 说明数据收集、存储、删除、导出规则
- 明确附件和个人信息的处理边界

## P1：上线后尽快补齐

这些事项不是首发阻断项，但会直接影响可用性、留存和运营效率。

### 1. 导入识别能力增强

- 优化 OCR 和 PDF 文本提取准确率
- 根据不同凭证类型建立解析模板
- 为低置信度字段增加强提醒和人工复核流程
- 保留原始文本与识别结果，便于追溯

### 2. 报表与导出

- 增加按对象、分类、时间段的统计报表
- 支持 Excel / CSV 导出
- 支持打印和对账单生成
- 增加月报、季报、年报视图

### 3. 提醒与通知

- 到期提醒
- 逾期提醒
- 计划还款提醒
- 支持站内通知、邮件或短信通知

### 4. 协作流程

- 支持多人查看和维护
- 支持备注、评论和处理记录
- 支持审批、复核或确认流程
- 支持负责人分配

### 5. 用户体验完善

- 首次使用引导
- 空状态、错误状态、导入失败说明
- 大量数据场景下的列表性能优化
- 更完整的移动端适配

## P2：形成产品竞争力

这些事项用于把项目从“能用”升级到“值得付费购买”。

### 1. SaaS 产品能力

- 租户隔离
- 套餐体系
- 计费和订阅管理
- 发票和订单管理

### 2. 高级账务能力

- 利息、手续费、滞纳金模型
- 分币种汇率处理
- 自动对账和异常检测
- 批量导入、批量更新、批量导出

### 3. 智能化增强

- 更强的凭证识别与字段推荐
- 自动分类和标签建议
- 风险记录识别
- 还款计划智能建议

### 4. 企业级能力

- 单点登录
- 更细粒度的权限策略
- 审批流配置
- 数据留存策略与归档

## 推荐实施顺序

建议按下面顺序推进：

1. 先完成 `P0` 的后端、数据库、账号、权限、安全、附件体系
2. 再补 `P0` 的测试、监控、日志和环境管理
3. 上线试运行后，再进入 `P1` 的报表、通知、协作和识别增强
4. 最后根据商业目标投入 `P2` 的 SaaS 化和高级能力

## 推荐技术改造方向

结合当前项目体量，建议下一阶段至少完成这些结构升级：

- 前端：从单文件脚本改为模块化前端工程
- 后端：提供 REST API 或同等级服务接口
- 数据库：使用正式关系型数据库
- 附件：接入对象存储
- 鉴权：使用正式会话或 token 鉴权
- 监控：增加错误监控、访问日志和性能指标

## 最小可商用版本建议

如果目标是先做一个“可收费试运营版”，最小范围建议是：

- 账号登录
- 后端接口
- 数据库存储
- 附件上传
- 基础权限
- 债务记录增删改查
- 还款计划与付款记录
- 导出功能
- 审计日志
- 隐私政策和用户协议

完成这批以后，才能算进入“可控风险的商用试运行”阶段。
