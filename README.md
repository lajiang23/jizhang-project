# 记账项目

一个面向个人或小团队的债务记账前端原型，用来管理借款、欠款、凭证导入和还款计划。

在线地址：
[https://lajiang23.github.io/jizhang-project/](https://lajiang23.github.io/jizhang-project/)

## 项目简介

这个项目是一个纯静态前端应用，主要解决“谁欠谁多少钱、何时到期、已经还了多少、后续怎么还”这类债务记录问题。

当前版本以浏览器端原型为主，适合先验证流程和界面交互，不依赖后端数据库。所有记录默认保存在浏览器本地 `localStorage` 中。

## 核心功能

1. 债务工作台
   录入和管理债务记录，支持区分“我欠别人”和“别人欠我”两种方向。
2. 手动新增与编辑
   可填写对象、本金、币种、发生日期、到期日、状态、分类、标签、备注、附件名等字段。
3. 条件筛选
   支持按方向、状态、分类、到期范围和对象名称进行筛选。
4. 文件导入识别
   支持导入图片、PDF、Excel、CSV 等文件，提取文本后生成待确认草稿。
5. 导入草稿确认
   导入结果会展示识别摘要、风险提示和字段预填，用户确认后再入账。
6. 还款计划管理
   支持为单条债务创建手动计划或智能生成分期计划，并可逐期调整金额、日期和完成状态。
7. 实际还款记录
   已还金额会进入付款记录，便于对比“计划还款”和“实际还款”。
8. 年度账单视图
   提供按年汇总的月度统计、年度发生额、年度已还额、年度新增应收和年度新增应还。
9. 示例数据加载
   内置演示数据，便于首次打开时快速查看完整流程。

## 功能说明

### 1. 债务记录字段

- 借贷方向：我欠别人 / 别人欠我
- 对象：个人或公司名称
- 本金：债务金额
- 币种：CNY、USD、EUR、JPY
- 发生日期 / 到期日
- 状态：进行中、已结清、已逾期
- 一级分类 / 二级分类
- 标签
- 备注
- 附件名

### 2. 导入能力

- 图片：通过 `Tesseract.js` 做 OCR 文本识别
- PDF：通过 `pdfjs-dist` 提取前几页文字
- Excel / CSV：读取表格并提取前几行摘要
- 导入后不会直接入账，而是先生成草稿，减少误识别带来的脏数据

### 3. 还款计划

- 支持手动创建分期计划
- 支持根据起始日期、期数、频率、规则、目标每期金额智能生成计划
- 支持逐期标记已还
- 支持手动修正每期金额和日期

### 4. 年度汇总

年度账单页用于查看某一年的债务变化情况，包括：

- 月度发生额趋势
- 月度明细列表
- 年度新增应收
- 年度新增应还
- 年度已还总额

## 技术实现

项目不依赖构建工具，直接由静态文件组成：

- `index.html`：页面结构
- `styles.css`：界面样式
- `app.js`：业务逻辑、状态管理、导入解析、计划计算

使用到的前端依赖：

- `PapaParse`：CSV 解析
- `xlsx`：Excel 读取
- `Tesseract.js`：图片 OCR
- `pdfjs-dist`：PDF 文本提取

## 数据存储说明

- 所有数据默认保存在当前浏览器本地 `localStorage`
- 不同浏览器、不同设备之间不会自动同步
- 清空浏览器站点数据后，记录会丢失
- 当前版本没有登录、权限、多用户协作和云端备份

## 适用场景

- 个人借款和还款记录
- 家庭内部往来记账
- 小型经营周转登记
- 业务启动垫资或客户欠款记录
- 需要快速验证债务管理流程的前端原型

## 当前限制

- 仅前端存储，没有后端数据库
- OCR 和文本提取结果依赖原始文件质量，不能保证百分百准确
- 暂不支持多人实时协作
- 暂不支持账户体系、权限控制和服务端持久化

## 本地运行

进入 `页面` 目录后，可通过任意静态 HTTP 服务启动，例如：

```powershell
python -m http.server 8123 --bind 127.0.0.1
```

然后访问：

```text
http://127.0.0.1:8123/
```
